Team development using github

使用github进行团队协作开发

github是一种常用的团队协作开发工具。在此次Travis.CI持续集成练习中，我们使用了github来进行团队管理，以下是我们的一些使用心得。

Issues的使用

github中的issues功能除了追踪各种想法、提出bug等外，还能用于进行团队任务分配和管理。

示例


issue的左边为成员对该issue的修改记录，右侧栏中为对该issue的各种操作：

Assignees:

团队成员可以将这个任务assign给任何人，也可以通过unassign删除成员。

Lables:

团队成员可以为该issue添加各种自定义的标签，方便issues的管理。

Projects:

将该issue划归特定Project，具体作用在稍后说明。

Milestone

每个issue只属于一个Milestone，用于标记任务在整个项目中的进程

Projects的使用

Projects是github于2016年推出的项目管理工具，协助开发者在开发流程中整合项目管理，让开发者可以直接在GitHub程序代码储存库中管理工作流程。
如下图所示，Projects将issues分为To do, In progress, Done三种。每当一个新issue创建时便被自动归于To do类，成员可以直接在界面上进行拖拽，将特定issue放到In progress中。当该issue被关闭时，自动被归为Done类。
Projects面板可以让我们很方便地监控项目进度。

branches的使用

当多人协作时，为保证主分支代码的正确性以及支持并行开发，我们常常需要每个团队成员创建一个自己的分支，在其上进行操作。

branch的创建与维护

一些常用指令：

  1. git branch dev           //创建名为dev的分支
  2. git checkout dev       //切换到dev分支
  3. git merge dev           //将dev中的代码merge到当前分支中
  4. git branch –set -upstream branchName origin/branchName
    //建立本地分支与远程分支的关联，branchName为自己设定的分支名。本地分支若不推送到远程，则对他人不可见。
  5. git push origin dev   //将本地代码push到远程指定分支中

当git无法自动合并分支时，我们必须手动解决冲突，然后重新提交。

强制更新

有时候我们无法顺利从远程pull下来代码，此时如果本地代码没有做什么修改，可以考虑将远程仓库中的代码强行更新到本地，具体指令如下：

  1. git fetch –all
  2. git reset –hard origin/master       //origin/master为远程分支名
  3. git pull

执行完毕若没有发生错误，即更新完成。
